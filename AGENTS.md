# AGENTS.md — Guide for AI Agents

This document tells an AI agent (Claude, GPT, Gemini, etc.) everything needed to understand, run, and extend this project autonomously.

## What This Project Does

Ez-Money-Glitch is a quantitative trading research system. It:
1. Downloads historical stock price data (S&P 500, NASDAQ 100, Oslo Bors) from Yahoo Finance
2. Computes technical indicators (RSI, MACD, Bollinger, ATR, etc.) on that data
3. Runs a neural network (SwingTradeNet v6) to generate BUY/NEUTRAL/SELL signals
4. Backtests trading strategies against 13 years of historical data (walk-forward, no look-ahead bias)
5. Runs an autonomous research loop that discovers better strategy parameters
6. Presents results in a local web dashboard

## Tech Stack

- Python 3.10+
- PyTorch (neural network)
- pandas, numpy (data manipulation)
- yfinance (data download)
- scikit-learn (preprocessing)
- Chart.js (dashboard, no npm needed)
- All data is local — no external APIs beyond Yahoo Finance

cd Ez-money-glitch
python -m venv venv

## How to Run (in order of complexity)

```bash
# 1. Quick test — signals for one ticker (no training needed)
python run.py signals AAPL --no-nn

# 2. Backtest all strategies (downloads data, ~5-15 min)
python run.py backtest --tickers AAPL MSFT NVDA TSLA GOOGL

# 3. Open dashboard to see results
python run.py dashboard
# open http://localhost:8080/index.html

# 4. Train the neural network (~1hr on CPU, 10min on GPU)
python run.py train

# 5. Run auto-researcher (continuous, Ctrl+C to stop)
python run.py research --hours 4
```

## Project Layout (which files do what)

```

nn/
  model.py               SwingTradeNet v6 class definition (no external deps beyond torch)
  train.py               Downloads data, builds dataset, trains model, writes results/training_log.json
  signals.py             Loads trained model, runs inference, returns BUY/NEUTRAL/SELL per ticker
  indicators.py          55 technical indicators computed from OHLCV data
data/
  fetch.py               get_prices(ticker) -> DataFrame. Caches to data/cache/ohlcv/*.parquet
  universe.py            get_sp500(), get_nasdaq100(), get_oslo_top60() ticker lists
strategies/
  classic.py             Signal functions: rsi_signal, macd_signal, bollinger_signal, etc.
  advanced.py            Signal functions: breadth_momentum_signal, kalman_trend_signal, etc.
  backtest.py            run_all_strategies() -> writes results/backtest_results.json
researcher/
  auto_researcher.py     run_researcher() -> endless loop, writes results/researcher_status.json
dashboard/
  index.html             Hub page — reads results/*.json via fetch()
  training.html          Training curves — reads results/training_log.json
  strategies.html        Equity curves + rankings — reads results/backtest_results.json
  researcher.html        Researcher feed — reads results/researcher_status.json
  serve.py               python dashboard/serve.py -> http://localhost:8080
results/                 JSON output files (gitignored, generated by running the code)
models/                  Trained model files (gitignored, generated by training)
```

## Common Tasks for an Agent

**Add a strategy and test it:**
```bash
# Edit strategies/advanced.py, add to researcher/auto_researcher.py, then:
python run.py research --dry-run
```

**Run a quick backtest on specific tickers:**
```bash
python run.py backtest --tickers AAPL MSFT NVDA --years 5
```

**Check if the neural network is training well:**
```bash
# Look at results/training_log.json
# val_acc should be > 33% (random baseline for 3-class problem)
# train_loss - val_loss gap > 0.2 = overfitting
```

**Generate signals without a trained model:**
```bash
python run.py signals AAPL NVDA TSLA --no-nn
```

## Known Issues / Limitations

- NN currently achieves ~39% val accuracy vs 33% random baseline — still room to improve
- Training with --sp500 (500 tickers, 13yr) takes 2-3 hours on CPU, ~20 min on GPU
- The Hurst adaptive strategy is slow (O(n*lags) per bar) — skip for quick runs
- Oslo Bors data quality from Yahoo Finance is lower than US data
- Walk-forward backtesting with only 15 tickers gives noisy Sharpe estimates; use 30+ for reliable results

## Environment Variables (optional)

None required. All data is from public Yahoo Finance. If you want to use a GPU, just ensure CUDA is installed — the code auto-detects it.
